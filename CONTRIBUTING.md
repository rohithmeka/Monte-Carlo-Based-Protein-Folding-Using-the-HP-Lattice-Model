## Contributing

Thank you for your interest in contributing to this project.

### What You Can Contribute

* Improvements to optimization strategies (e.g., better move sets, convergence methods)
* Code refactoring and performance enhancements
* Visualization improvements
* Bug fixes
* Documentation updates

### Guidelines

* Keep the code clean, modular, and readable
* Do not break existing functionality
* Follow the current coding style
* Avoid adding unnecessary dependencies

### How to Contribute

1. Fork the repository
2. Create a new branch (`feature/your-feature-name`)
3. Commit your changes with clear messages
4. Push to your fork and submit a pull request

### Reporting Issues

If you find a bug or have suggestions, open an issue with:

* A clear description of the problem
* Steps to reproduce
* Expected vs actual behavior
